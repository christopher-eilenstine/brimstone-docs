Static Site CMS

Static Site CMS (Content Management System) is used to generate static sites for Buildertrend clients, including homebuilders and home remodelers. Such websites are used by clients for marketing to their own customers. Buildertrend provides this CMS in the form of an admin panel, where clients can edit their site content and integrate with other Buildertrend applications, such as contact forms for lead generation, a login page, and more.

This project is meant to replace the demo website that Buildertrend shows prospective clients. That website has integrations (as iframes) to the full Buildertrend application. Please note that the demo site has some features that should not be implemented, so check with the Buildertrend contact if any questions arise!

This project is based on the Buildertrend proof of concept, which they put together as a basic prototype.

Features

See the milestones document for more information.

  • Admin panel to edit basic JSON file that drives the site
  • API to save/retrieve JSON data
    • This should load and save under a given domain name
    • Our actual API integration/GCP integration will be mocked in this project
    • Make get/post endpoints, but you just load/save from the local file system
      • Make this as hacky as you want, it will be replaced!
    • Any image/document uploads will be saved to the server separately and a link will be included in the JSON
  • Build out initial templates in React (see Website Admin.pdf)
    • Home, Blank, Gallery, Showcase, Blog, Login, Contact
  • Example content for a generic template (some of these could be made into their own templates if desired)
    • About/Employee, Floor Plan, Testimonials, Services, Privacy Policy, Process
  • At least two themes should be available—theme name should be stored in JSON
    • Color variations supported by themes
    • Some basic design manipulation by theme
  • Integrations (these are just iframes)
    • Integration with the BT login form
    • Integration with BT leads (contact us screen)

Stretch Goals

Prerequisites
  • The most recent NodeJS v14 is required to install Static Site CMS
    • Check the NodeJS version with node -v
    • Check the NPM version with npm -v
    • Check the NPX version with npx -v
  • An active Internet connection is required for the API server

Installing Static Site CMS
  1. Download the newest source code release of Static Site CMS
  2. Unzip the project folder using a file archiver, such as 7-Zip
  3. Open the root of the spring21-uno-ssg-{version} folder in a terminal window
  4. Execute npm install to install to install all build dependencies

Using Static Site CMS

See the create-react-app document for React specific instructions.

Starting the Static Site CMS
  1. Open the root of the spring21-uno-ssg-{version} folder in a terminal window
  2. Execute npm start to start the development server
  3. Wait for the "you can now view static-site-cms in the browser" prompt to appear

Editing a Website with the Static Site CMS
  1. Open a browser to http://localhost:3000/cms/brimstone.ddns.net/?action=edit
  2. Click the edit (:pencil2:) icon at the bottom-left of the window to open the sidebar
    1. Update the Title field text, as desired
    2. Update the Body field text, as desired
    3. Click the Save button to commit all changes
      • Otherwise, click the Reset button to discard all changes
  3. Click the left chevron (❰) icon to close the sidebar
    • Reload the page to confirm that all changes were saved, if desired
